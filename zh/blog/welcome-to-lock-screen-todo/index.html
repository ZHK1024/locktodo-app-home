欢迎来到 锁屏待办

为什么我们想做一款更安静的 iPhone 效率工具。

2026年5月16日 • 2 分钟阅读 • 更新于 2026年5月16日

锁屏待办 的出发点很简单：在每一次分心之前，先让你看到下一件真正重要的事。

大多数效率工具的问题

很多任务应用只有在你解锁手机、打开 App、进入具体界面之后才真正有用。它们适合整理和规划，却不一定适合那些犹豫的一秒钟。

我们反复思考锁屏这个位置，是因为它天然存在于你的一天里。与其让它只是通知的入口，不如让它先提醒你该继续什么。

设计原则

锁屏应该像温和的提示，而不是缩小版控制台。

为什么是锁屏

你会在出门前、会议间隙、想到一件事的时候、甚至只是想确认下一步要做什么的时候看一眼锁屏。

如果这个位置只展示一件最重要的事，它比拥挤的首页更有帮助。

产品气质和阅读气质应该一致

我们希望产品本身和官网博客有同一种感觉：

  • 视觉噪音更少
  • 层级更克制
  • 留白足够让内容呼吸
  • 依赖 typography，而不是靠装饰制造高级感

一个很小的例子

const todayFocus = reminders.find((item) => item.priority === "highest");

if (todayFocus) {
  showOnLockScreen(todayFocus.title);
}

重点不是代码本身，而是约束感。很多更好的体验，往往从“决定不展示什么”开始。

后面的文章会继续围绕 iPhone 工作流、Apple Reminders、快捷指令自动化和轻量日计划展开。它们应该真正帮助用户建立更安静的系统，而不是只为了获取流量。
